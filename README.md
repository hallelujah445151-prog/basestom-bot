# Telegram Bot для управления заказами в зуботехнической лаборатории

## Структура проекта

- `src/` - исходный код
  - `database.py` - работа с SQLite базой данных
  - `services/` - бизнес-логика
    - `reference_manager.py` - управление реестрами
    - `message_processor.py` - ИИ обработка сообщений
    - `notification_service.py` - отправка уведомлений и напоминаний
    - `user_manager.py` - управление пользователями
    - `reminder_service.py` - проверка сроков выполнения заказов
    - `report_service.py` - сбор статистики и формирование отчетов
  - `handlers/` - обработчики команд и сообщений
    - `registration.py` - регистрация пользователей
    - `admin.py` - админ-панель диспетчера
    - `orders.py` - обработка заказов
    - `reports.py` - обработка запросов отчетов
  - `utils/` - утилиты и хелперы
    - `reminder_background.py` - фоновый процесс напоминаний
  - `config/` - конфигурации
  - `bot.py` - главный файл бота
- `data/` - база данных и реестры
   - `references.json` - реестры врачей, техников и видов работ
   - `orders.db` - база данных заказов (создается автоматически)
- `src/.env` - переменные окружения (токены)
- `requirements.txt` - зависимости Python
- `VPS_DEPLOYMENT.md` - инструкция по деплою на VPS
- `DEPLOYMENT_FILES.md` - список файлов для деплоя
- `supervisor.conf` - конфигурация Supervisor для автозапуска
- `deploy.sh` - скрипт автоматической установки
- `src/.env.example` - пример переменных окружения
- `AGENTS.md` - документация для AI агента
- `references.md` - требования к проекту (архивный файл)
- `REPLIT_DEPLOYMENT.md` - инструкция для Replit (архивный файл)

## Установка

### Вариант 1: Локальная установка

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

2. Создайте бота через @BotFather в Telegram и получите токен

3. Получите API ключ от OpenRouter:
    - Зарегистрируйтесь на https://openrouter.ai/
    - Получите API ключ в настройках аккаунта
    - Используется бесплатная модель Meta Llama 3 8B (без затрат)

4. Добавьте токены в `.env`:
```
BOT_TOKEN=ваш_токен_от_BotFather
OPENROUTER_API_KEY=ваш_ключ_от_OpenRouter
```

### Вариант 2: Развертывание на VPS (для продакшена)

Для постоянной работы бота используйте VPS с автозапуском через Supervisor.

📖 **Подробная инструкция:** [VPS_DEPLOYMENT.md](VPS_DEPLOYMENT.md)

📦 **Список файлов для деплоя:** [DEPLOYMENT_FILES.md](DEPLOYMENT_FILES.md)

**Кратко:**
1. Загрузите файлы на VPS
2. Создайте виртуальное окружение и установите зависимости
3. Настройте `.env` с токенами
4. Настройте Supervisor для автозапуска
5. Запустите бота через Supervisor

- `/report_technicians` - Отчет по техникам (для диспетчера)

## Реализованный функционал

### Этап 1 - Базовая структура ✅
- SQLite база данных с таблицами: пользователи, заказы
- Реестры врачей, техников и видов работ в JSON
- ИИ обработка текстовых сообщений через OpenRouter с бесплатной моделью Meta Llama 3 8B
- Базовая структура бота с командами

### Этап 2 - Управление пользователями ✅
- Регистрация пользователей через команду `/register` с выбором роли
- Админ-панель для диспетчера с командой `/admin`
- Просмотр списка всех пользователей `/admin -> Список пользователей`
- Добавление пользователей вручную `/admin -> Добавить пользователя`
- Смена роли для тестирования `/changerole` (позволяет тестировать разный функционал)
- Система ролей: dispatcher (диспетчер), technician (техник), doctor (врач)

### Этап 3 - Обработка заказов ✅
- Прием фото заказ-наряда от диспетчера через `/neworder`
- Прием текста с назначением (например: "Мороков циркон на винте 7шт")
- ИИ нормализация сообщений через OpenRouter с бесплатной моделью Meta Llama 3 8B (без затрат)
- Формирование корректного сообщения: "Техник: Мороков А.А. Вид работы: Цирконевая коронка на импланте. Количество: 7шт."
- Запись заказов в SQLite базу данных

### Этап 4 - Уведомления ✅
- Автоматическая отправка уведомлений технику при создании заказа
- Автоматическая отправка уведомлений врачу при создании заказа
- Уведомление диспетчеру о создании заказа
- Отправка фото заказ-наряда вместе с сообщениями
- Обработка случаев, когда пользователь не зарегистрирован в боте

### Этап 5 - Напоминания ✅
- Автоматическая проверка сроков выполнения заказов
- Фоновый процесс для регулярной проверки (каждые 5 минут)
- Отправка напоминаний в 10:00 утра, за 1 день до дедлайна
- Отправка напоминаний технику и диспетчеру
- Отслеживание отправленных напоминаний (без повторов)
- Автоматический запуск при старте бота

### Этап 6 - Отчетность ✅
- Сбор статистики по заказам/выполнениям
- Формирование отчетов по врачам: количество заказов, виды работ
- Формирование отчетов по техникам: количество заказов, общее количество единиц, виды работ
- Формирование отчетов по видам работ: популярность работ, общее количество
- Отчеты за период: выбор даты начала и конца, полная статистика
- Доступно только для диспетчера

## Тестирование

### Локально

1. Запустите бота: `python src/bot.py`
    - 🤖 Бот запущен
    - 🔄 Фоновый процесс напоминаний запущен (проверка каждые 5 минут)

### На VPS

1. Следуйте инструкции в [VPS_DEPLOYMENT.md](VPS_DEPLOYMENT.md)
2. После настройки бота будет работать автоматически через Supervisor
3. Управление ботом:
   ```bash
   sudo supervisorctl start basestom-bot      # Запуск
   sudo supervisorctl stop basestom-bot       # Остановка
   sudo supervisorctl restart basestom-bot    # Рестарт
   sudo supervisorctl status basestom-bot     # Статус
   sudo supervisorctl tail -f basestom-bot    # Логи
   ```

2. В Telegram выполните `/register` для саморегистрации

3. После регистрации диспетчер может использовать `/admin` для управления пользователями

4. Зарегистрируйте техников и врачей через бота

5. Диспетчер может создавать заказы через `/neworder`
   - Сначала отправьте фото заказ-наряда
   - Затем текст с назначением (например: "Мороков циркон на винте 7шт")
   - Бот автоматически нормализует сообщение и сохранит в БД
   - Техник получит уведомление с фото заказ-наряда
   - Врач получит уведомление с фото заказ-наряда
   - Диспетчер получит подтверждение создания заказа

6. Напоминания работают автоматически:
   - Фоновый процесс проверяет каждые 5 минут, но отправляет только в 10:00 утра
   - Если заказ должен быть готов 13.02.2026, напоминание придет 12.02.2026 в 10:00
   - Напоминание приходит ОДИН РАЗ (дублирующиеся не отправляются)
   - Бот проверяет, было ли уже отправлено напоминание для заказа
   - Напоминание отправляется технику и диспетчеру

7. Диспетчер может просматривать отчеты:
   - `/report_doctors` - отчет по всем врачам
   - `/report_technicians` - отчет по всем техникам
   - `/report_work_types` - отчет по видам работ
   - `/report_period` - отчет за выбранный период
     - Введите начальную дату (ДД.ММ.ГГГГ)
     - Введите конечную дату (ДД.ММ.ГГГГ)
     - Получите детальный отчет за период

## Как работают напоминания

**Пример:**
- Заказ создан с дедлайном: 13.02.2026
- Напоминание придет: 12.02.2026 в 10:00 утра
- Получатели: Техник и Диспетчер
- Повторные напоминания: НЕ отправляются

**Логика работы:**
1. Фоновый процесс проверяет каждые 5 минут
2. В 10:00 проверяет все заказы с дедлайном завтра
3. Для каждого заказа проверяет, было ли уже отправлено напоминание
4. Если НЕТ - отправляет и отмечает в БД
5. Если ДА - пропускает
6. В 10:01 уже не проверяет (пропускает до следующего дня)

## 🚀 Развертывание

### Для продакшена (VPS)

Для постоянной работы бота рекомендуем развернуть на VPS с автозапуском.

📖 **Инструкция по деплою:** [VPS_DEPLOYMENT.md](VPS_DEPLOYMENT.md)

📦 **Файлы для деплоя:** [DEPLOYMENT_FILES.md](DEPLOYMENT_FILES.md)

**Преимущества VPS:**
- ✅ Работает постоянно, не спит
- ✅ Автоматический запуск при перезагрузке сервера
- ✅ Масштабируемость
- ✅ Полный контроль над сервером

**Быстрый старт на VPS:**
```bash
# 1. Загрузите файлы на VPS
scp -r basestom/* user@your-vps-ip:/tmp/basestom-bot/

# 2. На VPS
ssh user@your-vps-ip
sudo mkdir -p /opt/basestom-bot
sudo mv /tmp/basestom-bot/* /opt/basestom-bot/
cd /opt/basestom-bot
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 3. Настройте .env
nano src/.env
# Добавьте токены

# 4. Настройте Supervisor
sudo cp supervisor.conf /etc/supervisor/conf.d/basestom-bot.conf
sudo sed -i "s/your_username/$(whoami)/g" /etc/supervisor/conf.d/basestom-bot.conf
sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl start basestom-bot
```
