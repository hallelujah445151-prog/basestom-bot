# КРАТКОЕ РУКОВОДСТВО ПО СЕТЕВОЙ УСТОЙЧИВОСТИ

## Что было сделано для улучшения работы при блокировках

### ✅ Реализовано в src/bot.py:

1. **Увеличены таймауты**
   - connect_timeout: 30 сек
   - read_timeout: 30 сек
   - write_timeout: 30 сек
   - pool_timeout: 30 сек

2. **Добавлен retry-механизм**
   - До 5 попыток соединения
   - Экспоненциальная выдержка: 10, 20, 30 сек

3. **Добавлено логирование**
   - Детальные логи с timestamp
   - Понятные сообщения об ошибках
   - Номер попытки и время ожидания

4. **Улучшена обработка ошибок**
   - Отдельная обработка TimedOut
   - Отдельная обработка NetworkError
   - Graceful shutdown при Ctrl+C

## Как запустить бота

### Обычный запуск
```bash
python src/bot.py
```

### С автоперезапуском (Windows Batch)
```bash
run_bot_autorestart.bat
```

### С автоперезапуском (PowerShell)
```bash
powershell -ExecutionPolicy Bypass -File run_bot_autorestart.ps1
```

### С сохранением логов
```bash
python src/bot.py > bot.log 2>&1
```

## Что делать при ошибках

### Ошибка: TimedOut
```
Решение: Ждать, bot автоматически перезапустится
Время: 10-30 секунд между попытками
Попыток: 5
```

### Ошибка: NetworkError
```
Решение: Проверить VPN и интернет
Действие: Bot автоматически перезапустится
```

### Bot постоянно падает
```
Решения:
1. Использовать более стабильный VPN
2. Развернуть на VPS вне РФ
3. Использовать прокси
4. Проверить скорость интернета
```


## Статистика попыток соединения

Бот покажет в логах:
```
Attempt 1/5 - Первая попытка
Attempt 2/5 - Вторая попытка (через 10 сек)
Attempt 3/5 - Третья попытка (через 20 сек)
Attempt 4/5 - Четвертая попытка (через 30 сек)
Attempt 5/5 - Последняя попытка (через 30 сек)
```

После 5 неудачных попыток - bot завершается.

## Альтернативные решения

### 1. VPS (Рекомендуется)
- Германия, Нидерланды, Финляндия
- Цена: $3-5 в месяц
- Преимущества: Стабильное соединение

### 2. Прокси
```
В .env добавить:
PROXY_TYPE=socks5
PROXY_ADDR=127.0.0.1
PROXY_PORT=1080
```

### 3. Другие VPN
- Shadowsocks
- V2Ray
- Trojan
- Обычные VPN часто не работают

### 4. Webhook
- Требует SSL сертификата
- Более надежно, но сложнее
- Требует домен
